# 💻 Практичне завдання 3: Laravel та Symfony API

Цей репозиторій містить виконання лабораторної/практичної роботи зі створення API за допомогою двох популярних PHP-фреймворків: **Laravel** та **Symfony**. Також включено колекції запитів для тестування створених ендпоінтів.

## 📁 Структура проєкту

- 📂 `laravel/` — бекенд-частина, реалізована на фреймворку Laravel.
- 📂 `symfony/` — бекенд-частина, реалізована на фреймворку Symfony.
- 📂 `postman/` — експортовані колекції Postman для перевірки працездатності API.
- 📂 `scr/` — директорія зі скріншотами результатів тестування (розділена на підпапки `laravel` та `symfony`).

---

## ⚙️ Встановлення та запуск

Для локального розгортання проєктів переконайтеся, що у вас встановлені PHP, [Composer](https://getcomposer.org/) та [Postman](https://www.postman.com/).

### 1. Налаштування Laravel API
Перейдіть у директорію Laravel, встановіть необхідні пакети та запустіть сервер:
```bash
cd laravel
composer install
cp .env.example .env
php artisan key:generate
php artisan serve
2. Налаштування Symfony API
Перейдіть у директорію Symfony, встановіть залежності та запустіть сервер:

Bash
cd symfony
composer install
symfony server:start
🛠️ Підтвердження реалізації бекенду на Laravel
Для розробки першої серверної частини проєкту було використано фреймворк Laravel. Для перевірки коректності роботи створеного застосунку було налаштовано та протестовано колекцію запитів у Postman. Усі ендпоінти повертають відповідні статус-коди та структуровані JSON-дані.

Тестування працездатності API (CRUD операції)
🟢 Отримання списку всіх записів (GET)
Запит до GET /api/recipes (чи відповідного ендпоінту) повертає повний масив об'єктів із бази даних. Статус відповіді: 200 OK.

🔵 Створення нового запису (POST)
Надсилання даних через POST успішно створює новий об'єкт у базі. Статус відповіді: 201 Created (або 200 OK).

🟡 Отримання детальної інформації за ID (GET)
Запит GET з ідентифікатором повертає конкретну сутність. Статус відповіді: 200 OK.

🟠 Оновлення існуючих даних (PATCH)
Запит PATCH успішно модифікує вказані поля. Статус відповіді: 200 OK.

🔴 Видалення запису (DELETE)
Запит DELETE успішно видаляє об'єкт із бази даних.

🛠️ Підтвердження реалізації бекенду на Symfony
Для другої частини завдання було створено аналогічний API за допомогою фреймворку Symfony. Результати тестування ендпоінтів наведені нижче.

Тестування працездатності API (CRUD операції)
🟢 Отримання списку всіх записів (GET)
Отримання масиву даних із бази. Статус відповіді: 200 OK.

🔵 Створення нового запису (POST)
Успішне додавання нової сутності. Статус відповіді: 201 Created (або 200 OK).

🟡 Отримання детальної інформації за ID (GET)
Отримання одного запису за переданим ідентифікатором. Статус відповіді: 200 OK.

🟠 Оновлення існуючих даних (PUT/PATCH)
Оновлення інформації про існуючий запис. Статус відповіді: 200 OK.

🔴 Видалення запису (DELETE)
Успішне видалення запису з бази даних за ID.

Висновок: Виконані тести повністю підтверджують успішну інтеграцію, працездатність маршрутів (routing) та взаємодію з базою даних (CRUD операції) у межах обох фреймворків — Laravel та Symfony.
